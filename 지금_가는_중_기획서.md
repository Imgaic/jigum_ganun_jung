# 교내 실시간 장소 정보 서비스 기획서

## 1. 서비스명

**지금 가는 중**

## 2. 기획 배경

학생들은 교내에서 공부할 곳, 쉴 곳, 식사할 곳을 찾을 때 실제 현장의 혼잡도나 이용 가능 여부를 바로 알기 어렵다. 강의 사이의 짧은 시간에는 직접 여러 장소를 돌아다니며 확인하기 부담스럽고, 이미 도착한 뒤 자리가 없거나 혼잡한 상황을 마주하면 시간과 동선이 낭비된다.

특히 도서관, 라운지, 식당, 휴게공간처럼 시간대별 이용률이 빠르게 변하는 공간은 정적인 안내 정보만으로는 충분하지 않다. 따라서 학생들이 직접 현장 정보를 간단히 공유하고, 다른 학생들이 그 정보를 빠르게 확인할 수 있는 실시간 교내 장소 정보 서비스가 필요하다.

## 3. 문제 정의

**어떻게 교내 공간에 대한 최신 정보를 학생들에게 효율적으로 제공할 수 있을까?**

현재 학생들이 겪는 주요 문제는 다음과 같다.

- 교내 공간의 혼잡도 정보를 실시간으로 확인하기 어렵다.
- 원하는 목적에 맞는 장소를 빠르게 찾기 어렵다.
- 직접 이동하기 전까지 해당 공간의 상황을 알 수 없다.
- 학생들이 알고 있는 현장 정보가 공유되지 않고 개인 경험으로만 남는다.
- 사용자 제보 기반 정보는 유용하지만 신뢰도와 유효기간 관리가 필요하다.

## 4. 서비스 개요

**지금 가는 중**은 학생들이 교내 공간의 실시간 정보를 제보하고, 다른 학생들이 목적·건물·층수·혼잡도 기준으로 필요한 장소 정보를 확인할 수 있는 참여형 교내 장소 정보 플랫폼이다.

서비스는 크게 두 가지 흐름으로 구성된다.

- **정보 수집**: 현장에 있는 학생이 GPS, 상세 위치, 혼잡도, 예상 체류시간 등을 입력한다.
- **정보 제공**: 장소 정보가 필요한 학생이 키워드 검색이나 챗봇 질문을 통해 적합한 공간을 추천받는다.

이를 통해 학생들은 이동 전 공간 상황을 예측할 수 있고, 학교는 학생들의 실제 이용 흐름을 반영한 공간 데이터를 확보할 수 있다.

## 5. 목표 사용자

주요 사용자는 교내 공간을 자주 이용하는 학생이다.

- 공강 시간에 공부할 공간을 찾는 학생
- 식당이나 카페의 혼잡도를 알고 싶은 학생
- 휴식 공간, 라운지, 빈 강의실 등을 찾는 학생
- 특정 건물 안에서 조용하거나 덜 붐비는 장소를 찾는 학생
- 이동 시간을 줄이고 싶은 학생

## 6. 핵심 가치

### 6.1 실시간성

학생들이 직접 입력한 현장 정보를 바탕으로 최신 혼잡도와 장소 상태를 제공한다.

### 6.2 양방향성

정보를 받는 사용자와 제공하는 사용자가 분리되지 않고, 누구나 상황에 따라 제보자이자 이용자가 될 수 있다.

### 6.3 탐색 효율성

목적, 건물, 층수, 혼잡도 등 다양한 기준으로 장소를 찾을 수 있어 불필요한 이동을 줄인다.

### 6.4 데이터 순환 구조

사용자가 정보를 입력하면 포인트를 받고, 그 정보가 다른 사용자에게 제공되며, 시간이 지나면 자동으로 만료된다. 이를 통해 정보가 오래 방치되지 않고 계속 갱신되는 구조를 만든다.

## 7. 주요 기능

### 7.1 실시간 장소 정보 제보

사용자는 현재 위치한 장소의 정보를 간단히 입력할 수 있다.

입력 항목은 다음과 같다.

- GPS 기반 위치 확인
- 건물 및 층수 선택
- 상세 위치 선택: 라운지, 식당, 열람실, 휴게공간 등
- 혼잡도 선택: 매우 적음, 적음, 보통, 많음, 매우 많음
- 예상 체류시간 선택

상세 위치는 사용자가 직접 길게 입력하지 않도록 키워드 검색과 선택형 UI를 중심으로 구성한다.

### 7.2 위치 검증

정보 신뢰도를 높이기 위해 사용자가 입력하려는 장소와 GPS 정보의 일치 여부를 먼저 확인한다. GPS가 해당 건물 또는 인근 위치와 크게 다를 경우, 제보 반영도를 낮추거나 추가 확인 절차를 둔다.

### 7.3 정보 유효기간 관리

장소 정보는 시간이 지나면 실제 상황과 달라질 수 있으므로 유효기간을 둔다.

- 사용자가 입력한 예상 체류시간을 기본 기준으로 삼는다.
- 장소별 특성에 따라 만료시간에 가중치를 적용한다.
- 식당처럼 회전율이 빠른 장소는 짧게, 열람실처럼 체류시간이 긴 장소는 길게 유지한다.
- 만료 시간이 가까워지면 사용자에게 이동 여부를 묻는 푸시 알림을 보낸다.

### 7.4 포인트 보상

사용자가 장소 정보를 제보하면 열람 포인트를 지급한다. 포인트는 정보 제공 참여를 유도하고, 서비스 내 정보 순환을 활성화하는 장치로 활용한다.

포인트 운영 방식은 다음과 같이 설계할 수 있다.

- 정보 제보 시 기본 포인트 지급
- 다른 사용자에게 유용하다고 평가받은 제보에 추가 포인트 지급
- 허위 정보 또는 반복적인 부정확 제보는 반영도 감소

### 7.5 키워드 기반 장소 탐색

사용자는 버튼 선택 또는 직접 입력을 통해 원하는 장소 정보를 찾을 수 있다.

예시 키워드는 다음과 같다.

- 목적: 공부, 휴식, 식사, 대화, 조용한 공간
- 건물: 310관, 309관 등
- 층수: 3층, 5층 등
- 혼잡도: 덜 붐비는 곳, 자리 많은 곳

서비스는 선택된 키워드에 맞는 장소 리스트를 제공하고, 각 장소의 최신 혼잡도와 정보 업데이트 시간을 함께 보여준다.

### 7.6 챗봇 기반 장소 추천

사용자는 자연어로 원하는 장소를 물어볼 수 있다.

예시 질문은 다음과 같다.

- "지금 공부할 건데 적당한 곳 있어?"
- "지금 310관 식당 혼잡도 보여줘."
- "조용하고 사람 적은 라운지 추천해줘."

챗봇은 질문에서 목적, 건물, 층수, 혼잡도 등의 키워드를 추출하고, 조건에 맞는 장소와 연관 장소를 리스트로 제공한다.

## 8. 사용자 흐름

### 8.1 정보 제보 흐름

1. 사용자가 앱을 실행한다.
2. 현재 위치 기반으로 건물을 확인한다.
3. 사용자가 상세 위치를 선택한다.
4. 혼잡도를 1~5단계로 입력한다.
5. 예상 체류시간을 선택한다.
6. 제보가 등록되고 포인트가 지급된다.
7. 체류시간 종료 5분 전 푸시 알림으로 이동 여부를 확인한다.
8. 응답 결과에 따라 정보가 유지되거나 만료된다.

### 8.2 정보 탐색 흐름

1. 사용자가 목적, 건물, 층수 등의 조건을 선택하거나 직접 검색한다.
2. 서비스가 조건에 맞는 장소 리스트를 보여준다.
3. 사용자는 혼잡도, 최근 업데이트 시간, 상세 위치를 비교한다.
4. 원하는 장소를 선택해 상세 정보를 확인한다.
5. 필요하면 길찾기 또는 즐겨찾기로 이어진다.

## 9. 데이터 신뢰도 보완 방안

사용자 제보 기반 서비스에서 가장 중요한 요소는 정보의 신뢰성이다. 이를 위해 다음과 같은 방식을 적용한다.

### 9.1 다중 제보 평균

같은 장소에 여러 사용자가 제보한 혼잡도 값을 단순히 하나만 보여주지 않고, 시간과 신뢰도를 고려한 평균값으로 제공한다.

### 9.2 시간 가중치

최근 제보일수록 더 높은 비중을 부여하고, 오래된 제보는 자동으로 영향력을 낮춘다.

### 9.3 위치 일치도 반영

GPS 정보와 입력 장소가 일치할수록 제보 반영도를 높인다. 위치가 불명확한 경우에는 반영도를 낮추거나 참고 정보로만 활용한다.

### 9.4 사용자 신뢰도

정확한 제보를 꾸준히 한 사용자의 정보는 더 높은 가중치를 갖도록 설계한다. 반대로 부정확한 제보가 반복되면 반영도를 낮춘다.

### 9.5 만료 시스템

모든 장소 정보에는 유효기간을 둔다. 만료된 정보는 리스트에서 제외하거나 "업데이트 필요" 상태로 표시한다.

## 10. MVP 범위

초기 버전에서는 기능을 과도하게 확장하기보다 핵심 문제를 검증하는 데 집중한다.

MVP에 포함할 기능은 다음과 같다.

- 교내 주요 건물 및 공간 데이터 등록
- GPS 기반 건물 확인
- 상세 위치 선택
- 5단계 혼잡도 제보
- 제보 시간 기준 자동 만료
- 목적·건물·층수 기반 검색
- 장소별 혼잡도 리스트 제공
- 제보 보상 포인트 지급

MVP 이후 확장할 기능은 다음과 같다.

- 챗봇 기반 자연어 검색
- 사용자 신뢰도 시스템
- 장소별 만료시간 자동 조정
- 푸시 알림 기반 이동 여부 확인
- 개인화 추천
- 즐겨찾기 및 자주 가는 장소 저장

## 11. 기대효과

### 11.1 학생 편의성 향상

학생들은 이동 전에 공간의 혼잡도를 확인할 수 있어 시간을 절약하고 더 적합한 장소를 선택할 수 있다.

### 11.2 교내 공간 이용 효율화

특정 공간에만 사람이 몰리는 현상을 완화하고, 상대적으로 덜 알려진 공간의 활용도를 높일 수 있다.

### 11.3 최신 정보 제공

사용자 제보와 자동 만료 시스템을 결합해 오래된 정보가 계속 노출되는 문제를 줄일 수 있다.

### 11.4 참여형 데이터 구축

학생들이 직접 만든 데이터가 다시 학생들에게 도움이 되는 구조를 형성할 수 있다.

## 12. 고려사항

### 12.1 사용자 입력 부담

정보 수집 과정이 복잡하면 참여율이 낮아질 수 있다. 따라서 입력 항목을 최소화하고 선택형 UI를 중심으로 설계해야 한다.

### 12.2 허위 정보 가능성

제보 기반 서비스 특성상 부정확하거나 의도적으로 잘못된 정보가 입력될 수 있다. GPS 검증, 다중 제보 평균, 사용자 신뢰도 등을 통해 보완해야 한다.

### 12.3 정보 최신성 유지

혼잡도 정보는 빠르게 변하기 때문에 장소별 만료시간과 푸시 알림을 활용해 데이터가 계속 갱신되도록 해야 한다.

### 12.4 초기 데이터 부족

서비스 초반에는 제보 수가 부족할 수 있다. 특정 건물이나 공간을 중심으로 시범 운영하고, 포인트 보상으로 참여를 유도하는 전략이 필요하다.

## 13. 성공 지표

서비스의 성과는 다음 지표로 측정할 수 있다.

- 일간 장소 정보 제보 수
- 장소 정보 조회 수
- 제보 후 실제 조회로 이어진 비율
- 사용자 재방문율
- 제보 정보 만료 전 갱신 비율
- 사용자 만족도
- 추천 장소 선택 후 이동 완료율

## 14. 향후 확장 방향

향후에는 단순 혼잡도 제공을 넘어 개인 맞춤형 교내 공간 추천 서비스로 확장할 수 있다.

- 시간표와 연동한 다음 수업 근처 장소 추천
- 선호도 기반 조용한 공간 추천
- 친구와 함께 머무를 수 있는 공간 추천
- 교내 행사, 공사, 출입 제한 정보 연동
- 학교 공식 공간 데이터와 사용자 제보 데이터 결합

## 15. 결론

**지금 가는 중**은 학생들이 교내 공간을 더 효율적으로 이용할 수 있도록 돕는 실시간 장소 정보 서비스이다. 학생들의 현장 제보를 기반으로 최신 정보를 수집하고, 목적과 위치에 맞는 장소 정보를 제공함으로써 이동 낭비를 줄이고 교내 공간 활용도를 높일 수 있다.

핵심은 사용자가 부담 없이 정보를 제공하도록 만드는 간단한 입력 구조와, 제보 정보의 신뢰성을 유지하기 위한 데이터 관리 방식이다. 초기에는 주요 건물과 혼잡도 제보 기능을 중심으로 MVP를 구축하고, 이후 챗봇 검색과 개인화 추천 기능으로 확장하는 방향이 적합하다.
