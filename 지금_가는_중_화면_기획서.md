# 지금 가는 중 화면 기획서

## 1. 문서 개요

본 문서는 교내 실시간 장소 정보 서비스 **지금 가는 중**의 화면 구조와 주요 사용자 흐름을 정의하기 위한 화면 기획서이다.  
초기 서비스는 모바일 앱을 기준으로 설계하며, 사용자가 빠르게 장소 정보를 확인하고 부담 없이 현장 정보를 제보하는 것을 핵심 목표로 한다.

## 2. 서비스 핵심 흐름

서비스의 핵심 흐름은 두 가지이다.

1. **장소 정보 확인**
   - 사용자가 공부, 식사, 휴식 등 목적을 선택하거나 검색어를 입력한다.
   - 조건에 맞는 교내 장소 리스트를 확인한다.
   - 장소별 혼잡도, 최근 업데이트 시간, 상세 위치를 비교한다.

2. **장소 정보 제보**
   - 사용자가 현재 위치 또는 선택한 장소의 정보를 입력한다.
   - 혼잡도와 예상 체류시간을 선택한다.
   - 제보가 등록되고 포인트를 받는다.

## 3. 전체 화면 목록

| 구분 | 화면명 | 설명 | MVP 포함 여부 |
|---|---|---|---|
| 메인 | 홈 화면 | 검색, 추천, 제보 진입이 가능한 시작 화면 | 포함 |
| 탐색 | 장소 검색 화면 | 목적, 건물, 층수 등 조건 선택 | 포함 |
| 탐색 | 장소 리스트 화면 | 조건에 맞는 장소 목록 제공 | 포함 |
| 탐색 | 장소 상세 화면 | 특정 장소의 혼잡도와 상세 정보 제공 | 포함 |
| 제보 | 위치 확인 화면 | GPS 기반 현재 건물 확인 | 포함 |
| 제보 | 장소 정보 입력 화면 | 상세 위치, 혼잡도, 체류시간 입력 | 포함 |
| 제보 | 제보 완료 화면 | 포인트 지급 및 정보 등록 완료 안내 | 포함 |
| 사용자 | 포인트 화면 | 제보 포인트와 활동 내역 확인 | 확장 |
| 알림 | 이동 여부 확인 화면 | 체류시간 종료 전 푸시 알림 응답 | 확장 |
| AI | 챗봇 질문 화면 | 자연어 기반 장소 추천 | 확장 |

## 4. 정보 구조

```text
홈
├─ 장소 검색
│  ├─ 조건 선택
│  ├─ 장소 리스트
│  └─ 장소 상세
├─ 정보 제보
│  ├─ 위치 확인
│  ├─ 장소 정보 입력
│  └─ 제보 완료
├─ 챗봇 질문
│  └─ 추천 장소 리스트
└─ 마이페이지
   ├─ 포인트
   └─ 제보 내역
```

## 5. 공통 UI 원칙

- 화면 하단에는 주요 이동을 위한 탭 바를 배치한다.
- 사용자의 입력 부담을 줄이기 위해 직접 입력보다 버튼, 칩, 슬라이더, 선택형 UI를 우선 사용한다.
- 혼잡도는 숫자보다 직관적인 단계 표현을 사용한다.
- 장소 정보에는 반드시 최근 업데이트 시간을 함께 표시한다.
- 만료가 임박했거나 신뢰도가 낮은 정보는 별도 상태로 표시한다.

## 6. 화면별 와이어프레임 및 기능 정의

## 6.1 홈 화면

### 화면 목적

사용자가 앱을 실행했을 때 가장 빠르게 장소를 찾거나 정보를 제보할 수 있도록 한다.

### 와이어프레임

```text
┌─────────────────────────┐
│ 지금 가는 중             │
│                         │
│ ┌─────────────────────┐ │
│ │ 어디가 궁금하세요?   │ │
│ │ 공부할 곳, 310관... │ │
│ └─────────────────────┘ │
│                         │
│ 빠른 목적 선택           │
│ [공부] [식사] [휴식] [조용한 곳] │
│                         │
│ 현재 위치 근처           │
│ ┌─────────────────────┐ │
│ │ 310관 3층 라운지     │ │
│ │ 혼잡도 보통 · 3분 전 │ │
│ └─────────────────────┘ │
│ ┌─────────────────────┐ │
│ │ 중앙도서관 열람실    │ │
│ │ 혼잡도 많음 · 5분 전 │ │
│ └─────────────────────┘ │
│                         │
│ ┌─────────────────────┐ │
│ │ + 지금 장소 제보하기 │ │
│ └─────────────────────┘ │
│                         │
│ [홈] [검색] [제보] [내 정보] │
└─────────────────────────┘
```

### 주요 구성 요소

| 요소 | 설명 |
|---|---|
| 검색 입력창 | 장소명, 목적, 건물명을 입력할 수 있는 진입점 |
| 빠른 목적 선택 | 공부, 식사, 휴식 등 자주 쓰는 목적 버튼 |
| 현재 위치 근처 리스트 | GPS 기준 주변 장소의 최신 혼잡도 표시 |
| 제보 CTA | 정보 제보 흐름으로 이동하는 주요 버튼 |
| 하단 탭 바 | 홈, 검색, 제보, 내 정보 이동 |

### 주요 동작

- 검색창 선택 시 장소 검색 화면으로 이동한다.
- 빠른 목적 버튼 선택 시 해당 조건이 적용된 장소 리스트 화면으로 이동한다.
- 장소 카드 선택 시 장소 상세 화면으로 이동한다.
- 제보 버튼 선택 시 위치 확인 화면으로 이동한다.

## 6.2 장소 검색 화면

### 화면 목적

사용자가 원하는 장소 조건을 빠르게 선택할 수 있도록 한다.

### 와이어프레임

```text
┌─────────────────────────┐
│ 장소 찾기                │
│                         │
│ ┌─────────────────────┐ │
│ │ 검색어 입력          │ │
│ └─────────────────────┘ │
│                         │
│ 목적                    │
│ [공부] [식사] [휴식] [대화] │
│                         │
│ 건물                    │
│ [310관] [309관] [도서관] [전체] │
│                         │
│ 층수                    │
│ [1층] [2층] [3층] [4층 이상] │
│                         │
│ 혼잡도                  │
│ [여유] [보통] [혼잡 제외] │
│                         │
│ ┌─────────────────────┐ │
│ │ 조건에 맞는 장소 보기 │ │
│ └─────────────────────┘ │
│                         │
│ [홈] [검색] [제보] [내 정보] │
└─────────────────────────┘
```

### 주요 구성 요소

| 요소 | 설명 |
|---|---|
| 검색어 입력 | 사용자가 직접 장소명 또는 목적 입력 |
| 목적 칩 | 공부, 식사, 휴식 등 목적 선택 |
| 건물 칩 | 주요 건물 선택 |
| 층수 칩 | 층수 조건 선택 |
| 혼잡도 필터 | 원하는 혼잡도 범위 선택 |
| 결과 보기 버튼 | 선택 조건으로 리스트 조회 |

### 주요 동작

- 조건은 복수 선택이 가능하다.
- 검색어와 선택 조건을 함께 적용할 수 있다.
- 조건이 없는 경우 전체 장소 리스트를 보여준다.

## 6.3 장소 리스트 화면

### 화면 목적

조건에 맞는 장소를 비교하고 빠르게 선택할 수 있도록 한다.

### 와이어프레임

```text
┌─────────────────────────┐
│ 공부 · 310관             │
│ 8개 장소                 │
│                         │
│ 정렬 [가까운 순 ▼]        │
│                         │
│ ┌─────────────────────┐ │
│ │ 310관 3층 라운지     │ │
│ │ 보통 · 2분 전         │ │
│ │ 자리 있음 가능성 높음 │ │
│ └─────────────────────┘ │
│ ┌─────────────────────┐ │
│ │ 310관 5층 휴게공간   │ │
│ │ 적음 · 7분 전         │ │
│ │ 조용한 편             │ │
│ └─────────────────────┘ │
│ ┌─────────────────────┐ │
│ │ 310관 지하 식당      │ │
│ │ 많음 · 1분 전         │ │
│ │ 대기 가능성 있음      │ │
│ └─────────────────────┘ │
│                         │
│ [홈] [검색] [제보] [내 정보] │
└─────────────────────────┘
```

### 주요 구성 요소

| 요소 | 설명 |
|---|---|
| 검색 조건 요약 | 현재 적용된 목적, 건물, 조건 표시 |
| 결과 개수 | 조건에 맞는 장소 수 표시 |
| 정렬 메뉴 | 가까운 순, 여유로운 순, 최신 업데이트 순 |
| 장소 카드 | 장소명, 혼잡도, 업데이트 시간, 간단 설명 표시 |

### 주요 동작

- 장소 카드를 선택하면 장소 상세 화면으로 이동한다.
- 정렬 조건 변경 시 리스트 순서가 즉시 변경된다.
- 오래된 정보는 "업데이트 필요" 상태로 표시한다.

## 6.4 장소 상세 화면

### 화면 목적

특정 장소의 현재 상태를 자세히 확인하고, 사용자가 실제 이동 여부를 판단할 수 있도록 한다.

### 와이어프레임

```text
┌─────────────────────────┐
│ < 310관 3층 라운지       │
│                         │
│ 현재 혼잡도              │
│ ┌─────────────────────┐ │
│ │ 보통                 │ │
│ │ 최근 업데이트 2분 전  │ │
│ │ 신뢰도 높음           │ │
│ └─────────────────────┘ │
│                         │
│ 상세 정보                │
│ 위치: 310관 3층 엘리베이터 옆 │
│ 목적: 공부, 휴식          │
│ 예상 자리: 일부 있음       │
│                         │
│ 최근 제보                │
│ - 보통 · 2분 전           │
│ - 적음 · 6분 전           │
│ - 보통 · 9분 전           │
│                         │
│ ┌─────────────────────┐ │
│ │ 이 장소 정보 제보하기 │ │
│ └─────────────────────┘ │
└─────────────────────────┘
```

### 주요 구성 요소

| 요소 | 설명 |
|---|---|
| 장소명 | 건물, 층수, 상세 위치 표시 |
| 현재 혼잡도 카드 | 최신 혼잡도와 업데이트 시간 표시 |
| 신뢰도 상태 | 제보 수, GPS 일치도, 최신성을 바탕으로 표시 |
| 상세 정보 | 위치 설명, 목적 태그, 예상 자리 상태 |
| 최근 제보 | 최근 사용자 입력 내역 일부 표시 |
| 제보 버튼 | 같은 장소 정보 제보 화면으로 이동 |

### 주요 동작

- 정보가 오래된 경우 제보 버튼을 강조한다.
- 같은 장소에 대한 제보가 많을수록 신뢰도 상태를 높게 표시한다.
- 장소 제보 버튼 선택 시 장소가 미리 선택된 상태로 제보 화면에 진입한다.

## 6.5 위치 확인 화면

### 화면 목적

사용자가 실제 위치와 제보 장소가 일치하는지 확인해 데이터 신뢰도를 높인다.

### 와이어프레임

```text
┌─────────────────────────┐
│ 위치 확인                │
│                         │
│ 현재 감지된 위치          │
│ ┌─────────────────────┐ │
│ │ 310관 근처로 보여요   │ │
│ │ GPS 확인 완료         │ │
│ └─────────────────────┘ │
│                         │
│ 제보할 건물이 맞나요?    │
│ [310관] [309관] [직접 선택] │
│                         │
│ ┌─────────────────────┐ │
│ │ 다음                 │ │
│ └─────────────────────┘ │
└─────────────────────────┘
```

### 주요 구성 요소

| 요소 | 설명 |
|---|---|
| 현재 감지 위치 | GPS 기반 예상 건물 표시 |
| 건물 선택 버튼 | 감지된 건물 또는 다른 건물 선택 |
| 직접 선택 | GPS가 부정확할 때 수동 선택 |
| 다음 버튼 | 장소 정보 입력 화면으로 이동 |

### 주요 동작

- GPS 위치와 선택 건물이 일치하면 일반 제보로 처리한다.
- GPS와 선택 건물이 다르면 낮은 신뢰도 제보로 저장하거나 추가 확인을 요청한다.

## 6.6 장소 정보 입력 화면

### 화면 목적

사용자가 현장 정보를 빠르게 입력하고 제보할 수 있도록 한다.

### 와이어프레임

```text
┌─────────────────────────┐
│ 장소 정보 제보           │
│                         │
│ 건물                    │
│ [310관]                  │
│                         │
│ 상세 위치                │
│ ┌─────────────────────┐ │
│ │ 라운지, 식당 검색     │ │
│ └─────────────────────┘ │
│ [3층 라운지] [지하 식당] [5층 휴게공간] │
│                         │
│ 혼잡도                  │
│ [매우 적음] [적음] [보통] │
│ [많음] [매우 많음]       │
│                         │
│ 예상 체류시간            │
│ [15분] [30분] [1시간] [2시간] │
│                         │
│ ┌─────────────────────┐ │
│ │ 제보 등록하기         │ │
│ └─────────────────────┘ │
└─────────────────────────┘
```

### 주요 구성 요소

| 요소 | 설명 |
|---|---|
| 건물 표시 | 이전 단계에서 선택한 건물 |
| 상세 위치 검색 | 키워드 검색으로 장소 선택 |
| 추천 상세 위치 | 자주 제보되는 장소 빠른 선택 |
| 혼잡도 선택 | 5단계 선택형 입력 |
| 예상 체류시간 | 정보 유효기간 계산에 활용 |
| 등록 버튼 | 제보 완료 처리 |

### 주요 동작

- 상세 위치와 혼잡도는 필수 입력값이다.
- 예상 체류시간은 MVP에서는 선택값으로 둘 수 있다.
- 등록 완료 시 포인트 지급 후 제보 완료 화면으로 이동한다.

## 6.7 제보 완료 화면

### 화면 목적

제보 등록 결과를 알려주고, 사용자의 지속적인 참여를 유도한다.

### 와이어프레임

```text
┌─────────────────────────┐
│ 제보 완료                │
│                         │
│ ┌─────────────────────┐ │
│ │ 정보가 등록됐어요     │ │
│ │ +10 포인트 적립       │ │
│ └─────────────────────┘ │
│                         │
│ 등록한 정보              │
│ 310관 3층 라운지         │
│ 혼잡도 보통              │
│ 유효시간 약 30분          │
│                         │
│ ┌─────────────────────┐ │
│ │ 주변 장소 보기        │ │
│ └─────────────────────┘ │
│ ┌─────────────────────┐ │
│ │ 홈으로 가기           │ │
│ └─────────────────────┘ │
└─────────────────────────┘
```

### 주요 구성 요소

| 요소 | 설명 |
|---|---|
| 완료 메시지 | 제보 등록 성공 안내 |
| 포인트 안내 | 지급 포인트 표시 |
| 등록 정보 요약 | 제보 장소, 혼잡도, 유효시간 표시 |
| 주변 장소 보기 | 정보 탐색으로 자연스럽게 연결 |
| 홈으로 가기 | 홈 화면 복귀 |

## 6.8 챗봇 질문 화면

### 화면 목적

사용자가 자연어로 장소 정보를 물어보고 추천 결과를 받을 수 있도록 한다.

### 와이어프레임

```text
┌─────────────────────────┐
│ 장소 추천 챗봇           │
│                         │
│ ┌─────────────────────┐ │
│ │ 지금 공부할 곳 찾고   │ │
│ │ 있나요?              │ │
│ └─────────────────────┘ │
│                         │
│ 추천 질문                │
│ [공부할 곳 추천해줘]      │
│ [310관 식당 혼잡도]       │
│ [사람 적은 휴식 공간]     │
│                         │
│ 대화 영역                │
│ 사용자: 310관 공부할 곳 있어? │
│ 챗봇: 3층 라운지와 5층 휴게공간이 적당해요. │
│                         │
│ ┌─────────────────────┐ │
│ │ 메시지 입력          │ │
│ └─────────────────────┘ │
└─────────────────────────┘
```

### 주요 구성 요소

| 요소 | 설명 |
|---|---|
| 추천 질문 버튼 | 자주 묻는 질문을 빠르게 입력 |
| 대화 영역 | 사용자 질문과 추천 응답 표시 |
| 장소 추천 카드 | 답변 내 장소 리스트 카드 제공 |
| 메시지 입력창 | 자연어 질문 입력 |

### 주요 동작

- 사용자 질문에서 목적, 건물, 층수, 혼잡도 키워드를 추출한다.
- 정확히 일치하는 장소와 연관 장소를 함께 추천한다.
- 추천 장소 카드를 선택하면 장소 상세 화면으로 이동한다.

## 6.9 이동 여부 확인 화면

### 화면 목적

사용자의 예상 체류시간이 끝나기 전에 현재 장소 정보가 아직 유효한지 확인한다.

### 와이어프레임

```text
┌─────────────────────────┐
│ 아직 310관 3층 라운지에 있나요? │
│                         │
│ 제보한 정보가 곧 만료돼요. │
│ 현재 상태를 알려주면      │
│ 정보가 더 정확해져요.     │
│                         │
│ ┌─────────────────────┐ │
│ │ 아직 있어요           │ │
│ └─────────────────────┘ │
│ ┌─────────────────────┐ │
│ │ 이동했어요            │ │
│ └─────────────────────┘ │
│ ┌─────────────────────┐ │
│ │ 혼잡도 다시 입력하기  │ │
│ └─────────────────────┘ │
└─────────────────────────┘
```

### 주요 구성 요소

| 요소 | 설명 |
|---|---|
| 확인 메시지 | 사용자가 제보한 장소 기준 질문 |
| 아직 있어요 | 기존 정보 유효시간 연장 |
| 이동했어요 | 기존 정보 만료 처리 |
| 다시 입력하기 | 혼잡도 갱신 입력 화면으로 이동 |

## 9. 화면 설계 시 고려사항

### 9.1 제보 과정 최소화

정보 수집이 서비스의 핵심이지만 입력 과정이 길면 사용자가 이탈할 수 있다. 따라서 제보 화면은 상세 위치, 혼잡도, 체류시간 정도로 단순하게 구성한다.

### 9.2 혼잡도 표현의 직관성

혼잡도는 1~5 숫자보다 "매우 적음, 적음, 보통, 많음, 매우 많음"처럼 사용자가 바로 이해할 수 있는 텍스트로 표시한다.

### 9.3 정보 신뢰도 노출

장소 상세 화면에서는 최근 업데이트 시간과 신뢰도 상태를 함께 보여줘 사용자가 정보를 판단할 수 있도록 한다.

### 9.4 최신 정보 갱신 유도

정보가 오래된 장소는 단순히 숨기지 않고 "업데이트 필요" 상태로 표시해 사용자의 제보를 유도한다.

### 9.5 검색과 제보의 연결

장소를 확인한 사용자가 바로 해당 장소를 제보할 수 있도록 상세 화면에 제보 버튼을 제공한다. 이를 통해 정보 소비가 다시 정보 생산으로 이어지게 한다.

## 11. 결론

**지금 가는 중**의 화면 설계는 장소 탐색과 정보 제보를 최대한 짧은 흐름으로 연결하는 데 초점을 둔다. 홈 화면에서는 검색과 제보를 동시에 강조하고, 장소 리스트와 상세 화면에서는 혼잡도와 최신성을 빠르게 판단할 수 있게 구성한다.

초기 MVP에서는 검색, 리스트, 상세, 제보 화면을 중심으로 구현하고, 이후 챗봇 추천과 푸시 알림 기반 정보 갱신 기능을 확장하는 방향이 적합하다.
